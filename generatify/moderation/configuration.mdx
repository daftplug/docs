### Per-Category Actions

Configure different actions for different toxicity types:

```
Settings ‚Üí Content Moderation ‚Üí Advanced Actions

Toxicity: Hold for Moderation
Severe Toxicity: Block
Identity Attack: Block
Insult: Hold for Moderation
Profanity: Warn Only
Threat: Block
Sexually Explicit: Hold for Moderation
```

<Info>
**Example Use Case:** Allow mild profanity (Warn Only) but block severe toxicity and threats automatically.
</Info>

## Custom Rules

### Keyword-Based Moderation

Add your own blocklist/allowlist:

<Tabs>
  <Tab title="Blocklist">
    ### Block Specific Words/Phrases
    
    **Configuration:**
    ```
    Settings ‚Üí Content Moderation ‚Üí Blocklist
    
    Add keywords (one per line):
    spam keyword
    promotional phrase
    competitor name
    offensive term not caught by AI
    ```
    
    **Action Options:**
    - Block immediately
    - Hold for moderation
    - Assign spam flag
    
    **Example Use Cases:**
    - Block competitor mentions
    - Prevent spam keywords ("buy now", "click here")
    - Add language-specific profanity
    - Block coded language AI misses
  </Tab>
  
  <Tab title="Allowlist">
    ### Always Allow Specific Words
    
    **Configuration:**
    ```
    Settings ‚Üí Content Moderation ‚Üí Allowlist
    
    Add keywords (one per line):
    damn (in non-profane context)
    hell (technical term)
    your brand name (that triggers false positive)
    ```
    
    **Behavior:**
    - Overrides AI detection
    - Bypasses blocklist
    - Comment passes even if flagged
    
    **Use Sparingly:** Too many allowlist entries reduce effectiveness
  </Tab>
  
  <Tab title="Regex Patterns">
    ### Advanced Pattern Matching
    
    **Configuration:**
    ```
    Settings ‚Üí Content Moderation ‚Üí Advanced ‚Üí Regex
    
    Add patterns:
    /\b\d{3}-\d{3}-\d{4}\b/ (Block phone numbers)
    /https?:\/\/[^\s]+/ (Block URLs)
    /\b[A-Z0-9._%+-]+@[A-Z0-9.-]+\.[A-Z]{2,}\b/i (Block emails)
    ```
    
    **Common Patterns:**
    - Phone numbers
    - Email addresses
    - URLs/links
    - Credit card numbers
    - Specific formatting patterns
  </Tab>
</Tabs>

## Moderation Workflow

### Auto-Moderation Rules

Set up automated workflows:

<AccordionGroup>
  <Accordion title="New User Comments" icon="user-plus">
    **First-time commenters:**
    
    ```
    Settings ‚Üí Moderation Workflow ‚Üí New Users
    
    Action: Hold for moderation
    After X approved comments: Auto-approve future comments
    Recommended: 3 approved comments
    ```
    
    **Rationale:** Manually vet new users, then trust them
  </Accordion>
  
  <Accordion title="Link Moderation" icon="link">
    **Comments with URLs:**
    
    ```
    Settings ‚Üí Moderation Workflow ‚Üí Links
    
    Hold for moderation if comment contains:
    - 2+ links
    - Any links (for new users)
    - Links to blocked domains
    
    Auto-approve if:
    - Established user (5+ comments)
    - Link to your own domain
    ```
    
    **Prevents:** Spam, phishing, malicious links
  </Accordion>
  
  <Accordion title="Duplicate Detection" icon="copy">
    **Prevent spam/flooding:**
    
    ```
    Settings ‚Üí Moderation Workflow ‚Üí Duplicates
    
    Block if:
    - Exact same comment posted before
    - Similar comment (90%+ match) within 5 minutes
    - User posts X+ comments in Y minutes
    
    Example: Block if user posts 5+ comments in 1 minute
    ```
  </Accordion>
  
  <Accordion title="Length Restrictions" icon="text-width">
    **Enforce comment length:**
    
    ```
    Settings ‚Üí Moderation Workflow ‚Üí Length
    
    Minimum: 10 characters (prevent "ok", "lol")
    Maximum: 5,000 characters (prevent spam walls)
    
    Action on violation:
    - Show error message
    - Character counter for user guidance
    ```
  </Accordion>
</AccordionGroup>

## Best Practices

<AccordionGroup>
  <Accordion title="Start Conservative" icon="seedling">
    **Initial Setup:**
    1. Enable "Balanced" preset
    2. Set action to "Hold for Moderation" (not Block)
    3. Monitor for 1-2 weeks
    4. Review false positive rate
    5. Adjust thresholds based on data
  </Accordion>
  
  <Accordion title="Regular Review" icon="calendar-check">
    **Weekly Tasks:**
    - Review moderation queue
    - Approve false positives
    - Reject true positives
    - Note patterns
    - Adjust thresholds if FP rate > 20%
    
    **Monthly Tasks:**
    - Export analytics report
    - Analyze trends
    - Update blocklist/allowlist
    - Refine custom rules
  </Accordion>
  
  <Accordion title="Transparent Communication" icon="megaphone">
    **Inform your community:**
    
    - Publish community guidelines
    - Explain automated moderation
    - Set response time expectations
    - Provide appeal process
    - Link to guidelines in messages
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Too Many False Positives">
    **Solutions:**
    1. Increase thresholds by 10-20%
    2. Switch from Strict to Balanced
    3. Review specific triggering category
    4. Add common false positives to allowlist
    5. Whitelist established users
  </Accordion>
  
  <Accordion title="Toxic Comments Getting Through">
    **Solutions:**
    1. Lower thresholds (more sensitive)
    2. Switch to Strict preset
    3. Add specific terms to blocklist
    4. Enable "Block" action for severe categories
    5. Combine with human moderation
  </Accordion>
  
  <Accordion title="High Moderation Queue">
    **Solutions:**
    1. Increase thresholds (fewer holds)
    2. Enable auto-approve for trusted users
    3. Switch some categories to "Tag & Allow"
    4. Add more moderators
    5. Consider "Block" action for obvious toxicity
  </Accordion>
</AccordionGroup>
