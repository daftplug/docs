### Cache Configuration

<Tabs>
  <Tab title="Database Caching (Default)">
    **Built-in WordPress transients:**
    
    ```php
    // Configuration
    Settings → Advanced → Performance → Cache Method: Database
    
    // Pros:
    - No additional setup
    - Works everywhere
    - Simple management
    
    // Cons:
    - Slower than object cache
    - Database queries overhead
    - Not ideal for high traffic
    
    // Recommended for:
    - Small to medium sites (<10,000 pageviews/day)
    - Shared hosting
    - Simple setups
    ```
  </Tab>
  
  <Tab title="Redis (Recommended)">
    **Object caching with Redis:**
    
    ```php
    // Requirements:
    - Redis server
    - PHP Redis extension
    - Object cache plugin (Redis Object Cache)
    
    // Configuration:
    Settings → Advanced → Performance → Cache Method: Redis
    
    // Pros:
    - Very fast (in-memory)
    - Scales well
    - Low overhead
    
    // Cons:
    - Requires server setup
    - Additional cost
    
    // Recommended for:
    - Medium to large sites (>10,000 pageviews/day)
    - VPS or dedicated hosting
    - High performance needs
    ```
    
    **Setup:**
    ```bash
    # Install Redis
    sudo apt-get install redis-server
    
    # Install PHP extension
    sudo apt-get install php-redis
    
    # Install WordPress plugin
    wp plugin install redis-cache --activate
    ```
  </Tab>
  
  <Tab title="Memcached">
    **Object caching with Memcached:**
    
    ```php
    // Configuration:
    Settings → Advanced → Performance → Cache Method: Memcached
    
    // Similar to Redis:
    - In-memory caching
    - Fast performance
    - Requires setup
    
    // Choose Redis over Memcached unless:
    - Already using Memcached
    - Specific hosting requirements
    ```
  </Tab>
  
  <Tab title="File-Based">
    **File system caching:**
    
    ```php
    // Configuration:
    Settings → Advanced → Performance → Cache Method: File
    
    // Pros:
    - Faster than database
    - No additional services
    
    // Cons:
    - Slower than memory caching
    - File system overhead
    - Can fill disk space
    
    // Recommended for:
    - Sites without object cache
    - Better than database, worse than Redis/Memcached
    ```
  </Tab>
</Tabs>


**Programmatically:**

```php

// Clear all Generatify caches
do_action( 'generatify_clear_all_caches' );
```

## API Optimization

### Request Batching

Batch multiple operations to reduce API calls:

```php
// Instead of individual requests:
// ❌ Bad (3 API calls)
generate_tags( $post_id ); // 1 call
generate_categories( $post_id ); // 1 call
generate_meta_description( $post_id ); // 1 call

// Use batch processing:
// ✅ Good (1 API call)
generatify_batch_process( $post_id, [
    'tags',
    'categories',
    'meta_description',
]);
```

**Enable batching:**

```
Settings → Advanced → Performance
✅ Enable API Request Batching
```

**Savings:** 60-70% cost reduction for bulk operations

### Token Optimization

Reduce token usage to lower costs:

<AccordionGroup>
  <Accordion title="Shorter Prompts" icon="compress">
    **Optimize prompt length:**
    
    ```php
    // ❌ Verbose (100 tokens)
    "Please analyze the following blog post content and generate 
    a comprehensive list of relevant tags that accurately describe 
    the main topics, themes, and subjects discussed in the article. 
    Make sure the tags are SEO-friendly and commonly used..."
    
    // ✅ Concise (20 tokens)
    "Generate 5 SEO-friendly tags for this post:\n\n{content}"
    ```
    
    **Savings:** 80% token reduction
  </Accordion>
  
  <Accordion title="Content Truncation" icon="scissors">
    **Send only necessary content:**
    
    ```php
    // For tagging/categorization, send excerpt instead of full post
    
    // ❌ Full post (2000 tokens)
    $content = $post->post_content;
    
    // ✅ First 500 words (500 tokens)
    $content = wp_trim_words( $post->post_content, 500 );
    ```
    
    **Configuration:**
    ```
    Settings → Automations → Advanced
    Content Length: First 500 words (default: Full content)
    ```
    
    **Savings:** 75% token reduction (quality mostly maintained)
  </Accordion>
  
  <Accordion title="Response Length Limits" icon="ruler">
    **Limit response tokens:**
    
    ```php
    // Set max_tokens appropriate to task
    
    // Tags (5 tags * 3 tokens = ~15 tokens)
    max_tokens: 50
    
    // Meta description (~30 words = ~40 tokens)
    max_tokens: 60
    
    // Chat response (100-200 words = ~150-300 tokens)
    max_tokens: 400
    ```
    
    **Don't over-allocate:** Unused tokens still count against rate limits
  </Accordion>
  
  <Accordion title="Strip HTML" icon="code">
    **Remove unnecessary formatting:**
    
    ```php
    // Strip HTML tags before sending to AI
    $clean_content = wp_strip_all_tags( $post->post_content );
    
    // Remove extra whitespace
    $clean_content = preg_replace( '/\s+/', ' ', $clean_content );
    ```
    
    **Savings:** 10-20% token reduction
  </Accordion>
</AccordionGroup>

## Troubleshooting Performance

<AccordionGroup>
  <Accordion title="Slow Response Times">
    **Causes:**
    - No caching enabled
    - Slow AI provider
    - Large prompts/content
    - Database slow queries
    
    **Solutions:**
    1. Enable caching
    2. Switch to faster model (GPT-4o vs GPT-4)
    3. Truncate content to 500 words
    4. Install object cache
    5. Optimize database
  </Accordion>
  
  <Accordion title="High API Costs">
    **Causes:**
    - Using expensive models
    - No caching
    - Verbose prompts
    - No rate limiting
    
    **Solutions:**
    1. Switch to GPT-4o Mini
    2. Enable caching everywhere
    3. Optimize prompts
    4. Set rate limits
    5. Enable request batching
  </Accordion>
  
  <Accordion title="Cache Not Working">
    **Diagnostics:**
    ```
    Settings → Advanced → Performance → Cache Debug
    ✅ Enable cache debugging
    ```
    
    **Check:**
    1. Cache method configured correctly
    2. Object cache plugin active (if using Redis/Memcached)
    3. Sufficient disk space (file cache)
    4. Redis/Memcached service running
    5. Cache duration not set to 0
  </Accordion>
</AccordionGroup>
