## Setup Guide

### Step 1: Create OpenAI Account

<Steps>
  <Step title="Visit OpenAI Platform">
    Go to [platform.openai.com](https://platform.openai.com)
  </Step>

  <Step title="Sign Up">
    Create an account with email or Google/Microsoft sign-in
  </Step>

  <Step title="Verify Email">
    Check your inbox and verify your email address
  </Step>

  <Step title="Add Payment Method">
    Navigate to **Billing** and add a credit/debit card
  </Step>
</Steps>

### Step 2: Generate API Key

<Steps>
  <Step title="Navigate to API Keys">
    In OpenAI Platform, go to **API Keys** section
  </Step>

  <Step title="Create New Key">
    Click **Create new secret key**
  </Step>

  <Step title="Name Your Key">
    Give it a descriptive name (e.g., "Generatify WordPress Plugin")
  </Step>

  <Step title="Copy Key">
    Copy the key immediately—you won't see it again!
  </Step>
</Steps>

<img src="/images/openai-api-key.png" alt="Creating OpenAI API Key" />

<Warning>
  **Save your API key securely!** Once you close the modal, the key cannot be viewed again. You'll need to create a new one if lost.
</Warning>

### Step 3: Add to Generatify

<Steps>
  <Step title="Open WordPress Admin">
    Navigate to **Generatify → Settings → Integrations**
  </Step>

  <Step title="Find OpenAI Section">
    Scroll to the **OpenAI** integration block
  </Step>

  <Step title="Enable OpenAI">
    Toggle **Enable OpenAI** to ON
  </Step>

  <Step title="Paste API Key">
    Paste your OpenAI API key in the field
  </Step>

  <Step title="Save Settings">
    Click **Save Settings**
  </Step>

  <Step title="Verify Connection">
    Look for a green checkmark confirming successful connection
  </Step>
</Steps>

## Cost Management

### Understanding Token Pricing

**What are tokens?**
- Tokens ≈ words (1,000 tokens ≈ 750 words)
- Both input (prompt) and output (response) count
- Prices vary by model

**Example Calculation:**
```
Task: Generate 5 post tags
Input tokens: ~200 (post title + content excerpt)
Output tokens: ~20 (5 tags)

Cost with GPT-3.5 Turbo:
Input: 200 tokens × $0.50 / 1M = $0.0001
Output: 20 tokens × $1.50 / 1M = $0.00003
Total: ~$0.00013 per post
```

### Setting Budget Limits

<Steps>
  <Step title="Go to Billing Settings">
    In OpenAI Platform, navigate to **Billing → Usage limits**
  </Step>

  <Step title="Set Hard Limit">
    Set a maximum monthly spend (e.g., $50)
  </Step>

  <Step title="Set Soft Limit">
    Get email alerts at a threshold (e.g., $40)
  </Step>

  <Step title="Enable Email Alerts">
    Turn on notifications for usage milestones
  </Step>
</Steps>

Track your spending at [platform.openai.com/usage](https://platform.openai.com/usage):

- **Daily usage:** See costs by day

- **By feature:** Track usage patterns
- **Cost projections:** Estimated monthly total

## Best Practices

<AccordionGroup>
  <Accordion title="Choose the Right Model" icon="brain">
    Don't use GPT-4 for everything:
    - **GPT-4 Turbo:** Quality-critical tasks (chatbot, meta descriptions)
    - **GPT-3.5 Turbo:** Automations and high-volume tasks
    - **GPT-4o Mini:** Simple, repetitive tasks

    This can reduce costs by 90% without sacrificing much quality.
  </Accordion>

  <Accordion title="Optimize Prompts" icon="message">
    Shorter prompts = lower costs:
    - Be concise but clear
    - Remove unnecessary context
    - Use system instructions for reusable guidelines
    - Cache responses when possible
  </Accordion>

  <Accordion title="Monitor Usage Weekly" icon="chart-line">
    Check your usage dashboard regularly:
    - Spot unusual spikes
    - Identify cost-heavy features
    - Adjust model selection
    - Set alerts before hitting limits
  </Accordion>

  <Accordion title="Secure Your API Key" icon="lock">
    Protect your key:
    - Never commit to Git
    - Use environment variables
    - Rotate keys every 3-6 months
    - Revoke if compromised
    - Use separate keys for dev/prod
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Invalid API Key">
    **Error:** "Incorrect API key provided"

    **Solutions:**
    1. Check for typos or extra spaces
    2. Verify key is active in OpenAI dashboard
    3. Ensure billing is set up
    4. Generate a new key
  </Accordion>

  <Accordion title="Rate Limit Exceeded">
    **Error:** "Rate limit reached"

    **Solutions:**
    1. Wait a few seconds and retry
    2. Check your tier limits
    3. Upgrade to higher tier
    4. Reduce concurrent requests
    5. Implement exponential backoff
  </Accordion>

  <Accordion title="Insufficient Quota">
    **Error:** "You exceeded your current quota"

    **Solutions:**
    1. Add payment method if not set
    2. Check hard limit in billing settings
    3. Increase usage limit
    4. Wait for monthly reset
  </Accordion>

  <Accordion title="Model Not Found">
    **Error:** "Model not found"

    **Solutions:**
    1. Check model name spelling
    2. Verify you have access to the model
    3. Some models require waitlist approval
    4. Use a different model temporarily
  </Accordion>
</AccordionGroup>

## Advanced Configuration

### Using Environment Variables

For enhanced security, store API keys in environment variables:

```php
// wp-config.php
define('OPENAI_API_KEY', getenv('OPENAI_API_KEY'));
```

Then use a WordPress filter:

```php
add_filter('generatify_openai_api_key', function($key) {
    if (defined('OPENAI_API_KEY')) {
        return OPENAI_API_KEY;
    }
    return $key;
});
```

### Organization Keys

For teams, use organization-level API keys:

1. Create organization in OpenAI
2. Invite team members
3. Generate organization API key
4. Track usage by organization
