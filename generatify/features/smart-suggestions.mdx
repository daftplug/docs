## Smart Compose

Inline autocomplete that suggests text as users type in textareas‚Äîperfect for comments, contact forms, and review fields.

### How It Works

1. User starts typing in a textarea
2. Browser analyzes the text locally
3. Predictions appear in gray text
4. Press **Tab** or **‚Üí** to accept suggestion
5. Continue typing normally to ignore

- ‚ú® **Real-time predictions** as you type
- üîí **Privacy-first** - runs entirely in browser

- üì± **Mobile-friendly** with touch support
- üéØ **Context-aware** predictions

## Related Posts

Display AI-powered related content recommendations that keep readers engaged and reduce bounce rates.

### How It Works

1. Visitor views a post/page
2. Algorithm analyzes categories and tags
3. Finds semantically related content
4. Displays visually appealing recommendations
5. Updates dynamically per page

<img src="/images/related-posts-widget.png" alt="Related Posts Widget" />

### Key Features

- üéØ **Smart matching** based on taxonomy
- üìä **Performance optimized** with caching
- üé® **Customizable design** and layout
- üì± **Fully responsive** on all devices
- üîß **Developer-friendly** with hooks
- üöÄ **SEO-friendly** internal linking

### Display Options

<Tabs>
  <Tab title="Auto-Insert">
    Automatically append related posts after content:
    - Below post content (default)
    - Above content
    - Both above and below
    - Sidebar widget
  </Tab>

  <Tab title="Shortcode">

    ```shortcode

    ```

    With parameters:
    ```shortcode

            'quantity' => 6,

    ?>
    ```
  </Tab>

  <Tab title="Widget">
    Add to sidebar via **Appearance ‚Üí Widgets**:
    - Drag "Generatify Related Posts" widget
    - Configure title, quantity, layout
    - Save and preview

### Matching Algorithm

Related Posts uses intelligent taxonomy-based matching:

<Steps>
  <Step title="Primary Match">
    Find posts sharing the same **categories**
  </Step>

  <Step title="Secondary Match">
    If insufficient results, match by **tags**
  </Step>

  <Step title="Scoring">
    Rank by number of shared terms
  </Step>

  <Step title="Exclusion">
    Remove current post and duplicates
  </Step>

  <Step title="Limit">
    Return top N results (per settings)
  </Step>
</Steps>

<Info>
  **Performance:** Related posts are cached per post to minimize database queries. Cache refreshes automatically when content is updated.
</Info>

## Customization

### Smart Compose Styling

Customize appearance with CSS:

```css
/* Suggestion text color */
.generatify-smart-compose-suggestion {
    color: #999;
    font-style: italic;
}

/* Accepted suggestion animation */
.generatify-suggestion-accepted {
    animation: fadeIn 0.3s ease;
}
```

## Best Practices

<AccordionGroup>
  <Accordion title="Smart Compose Targeting" icon="crosshairs">
    Choose the right textareas:
    - ‚úÖ Comment forms (high engagement)
    - ‚úÖ Contact forms (helpful for users)
    - ‚úÖ Review fields (speed up feedback)
    - ‚ùå Admin forms (less beneficial)
    - ‚ùå Login/register (security concern)
  </Accordion>

  <Accordion title="Related Posts Quantity" icon="list-ol">
    Balance quantity with quality:
    - **3-4 posts:** Best for sidebars
    - **6-8 posts:** Good for post footers
    - **12+ posts:** Use carousel or paginated grid

    Too few = missed opportunities  
    Too many = overwhelming, slow page
  </Accordion>

  <Accordion title="Taxonomy Structure" icon="sitemap">
    Related Posts works best with:
    - Clear, distinct categories
    - Relevant, descriptive tags
    - Consistent taxonomy usage
    - Avoid over-tagging (5-10 tags max per post)
  </Accordion>

  <Accordion title="Visual Hierarchy" icon="layer-group">
    For Related Posts widget:
    - Use compelling featured images
    - Write concise, descriptive excerpts
    - Choose readable layouts
    - Ensure mobile responsiveness
    - Test different designs
  </Accordion>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Smart Compose not working">
    **Solutions:**
    1. Verify feature is enabled
    2. Check target textarea selector is correct
    3. Test in different browser
    4. Look for JavaScript errors in console
    5. Ensure minimum character threshold is met
  </Accordion>

  <Accordion title="Related Posts showing irrelevant content">
    **Solutions:**
    1. Review post categories and tags
    2. Ensure posts share taxonomy terms
    3. Increase quantity to see more options
    4. Switch matching logic (categories vs tags)
    5. Manually curate categories for better matching
  </Accordion>

  <Accordion title="Related Posts not appearing">
    **Solutions:**
    1. Feature enabled in settings?
    2. Current post has categories/tags?
    3. Other posts share the same terms?
    4. Clear cache (plugin, browser, server)
    5. Check post type is enabled in settings
  </Accordion>

  <Accordion title="Performance issues">
    **Solutions:**
    1. Enable object caching (Redis, Memcached)
    2. Reduce quantity of related posts
    3. Disable thumbnails to reduce image processing
    4. Use CDN for faster image delivery
    5. Enable lazy loading for images
  </Accordion>
</AccordionGroup>

## Developer Resources

### Hooks & Filters

Customize behavior with WordPress hooks:

```php
// Modify related posts query
add_filter('generatify_related_posts_args', function($args) {
    $args['posts_per_page'] = 10;
    return $args;
});

// Change related posts output
add_filter('generatify_related_posts_html', function($html, $posts) {
    // Custom HTML template
    return $custom_html;
}, 10, 2);

// Exclude specific post IDs
add_filter('generatify_related_posts_exclude', function($exclude_ids) {
    $exclude_ids[] = 123; // Exclude post ID 123
    return $exclude_ids;
});
```
