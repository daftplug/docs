## Feature Details

### ü§ñ AI Assistant Chatbot

<img src="/images/chatbot-feature.png" alt="AI Assistant Chatbot" />

Your personal WordPress copilot that understands your entire site's context.

**What it does:**
- Answers questions about your WordPress site
- Writes and rewrites content in your brand voice
- Explains plugin and theme errors in plain language
- Translates text and generates ideas
- Troubleshoots issues with step-by-step guidance

**Best for:** Content creators, site managers, developers

**Requirements:** AI provider API key (OpenAI, Anthropic, or Google)

### ‚öôÔ∏è Automations

<img src="/images/automations-feature.png" alt="AI Automations" />

Let AI handle repetitive content tasks automatically in the background.

**What it includes:**
- **Auto Post Tags** - Generates relevant tags for posts without them

### üí¨ Helpdesk Agent

<img src="/images/helpdesk-feature.png" alt="Helpdesk Agent" />

A front-end chat widget that provides instant support to your visitors 24/7.

<img src="/images/moderation-feature.png" alt="Content Moderation" />

Keep your community healthy by detecting toxic content before it's published.

## Enabling Features

All features can be toggled on/off independently:

1. Navigate to **Generatify** in your WordPress admin
2. Select the feature tab you want to configure
3. Toggle the feature to **ON**
4. Configure settings as needed
5. Click **Save Settings**

<Info>
  **Mix and Match:** You don't have to enable all features. Start with one or two that solve your immediate needs, then expand as you get comfortable.
</Info>

## Performance Impact

Generatify is built for performance:

<AccordionGroup>
  <Accordion title="Minimal Frontend Load" icon="gauge-high">
    - Most features run in admin area or backend
    - Frontend features load conditionally (only where needed)
    - Lightweight JavaScript with async loading
    - No jQuery dependencies for modern browsers
  </Accordion>

  <Accordion title="Efficient Backend Processing" icon="server">
    - Automations run on `save_post` hook (during publish, not on every page load)
    - AI calls are asynchronous where possible
    - Caching prevents duplicate API calls
    - Database queries are optimized with proper indexes
  </Accordion>

  <Accordion title="Resource Usage" icon="memory">
    - Plugin size: ~2MB (including vendor libraries)
    - Database footprint: Minimal (uses post meta and options)
    - No additional database tables required
    - Compatible with object caching (Redis, Memcached)
  </Accordion>
</AccordionGroup>
