## Overview

Automatic Post Tags uses AI to analyze your content and assign relevant, contextual tags automatically. Say goodbye to manual tagging and hello to consistent, comprehensive taxonomy.

## How It Works

<Steps>
  <Step title="Content Analysis">
    AI analyzes post title, content, and excerpt
  </Step>
  
  <Step title="Context Understanding">
    Identifies key topics, themes, and concepts
  </Step>
  
  <Step title="Existing Tag Review">
    Checks your existing WordPress tags for consistency
  </Step>
  
  <Step title="Tag Generation">
    Generates 3-10 relevant tags (configurable)
  </Step>
  
  <Step title="Automatic Assignment">
    Assigns tags to the post
  </Step>
</Steps>

<Info>
**Smart Detection:** AI can create new tags or use existing ones from your WordPress tag library.
</Info>

## Setup Guide

### Enable Auto Tags

<Steps>
  <Step title="Navigate to Settings">
    Go to **WordPress Admin → Generatify → Settings → Automations**
  </Step>
  
  <Step title="Find Auto Tags Section">
    Scroll to **Automatic Post Tags**
  </Step>
  
  <Step title="Enable Feature">
    Toggle **Enable Auto Tags** to ON
  </Step>
  
  <Step title="Configure Settings">
    Set your preferred options (see Configuration below)
  </Step>
  
  <Step title="Save Settings">
    Click **Save Settings**
  </Step>
</Steps>

## Trigger Options

Choose when tags are automatically generated:

<Tabs>
  <Tab title="On Publish">
    ### Automatic on Publish
    
    Tags are generated when you click **Publish** on a new post.
    
    **Pros:**
    - Most common workflow
    - Tags ready immediately
    - No manual action needed
    
    **Cons:**
    - Slight delay before post is live
    - Uses AI credit on every publish
    
    **Best For:** Standard blogging workflow
  </Tab>
  
  <Tab title="On Save Draft">
    ### Automatic on Save
    
    Tags generated whenever you save a draft.
    
    **Pros:**
    - Tags ready before publishing
    - Can review/edit before going live
    - Preview how tags will look
    
    **Cons:**
    - Uses AI credit on every save
    - Tags may regenerate multiple times
    
    **Best For:** Perfectionist workflow
  </Tab>
  
  <Tab title="Manual Trigger">
    ### Manual Button Click
    
    Generate tags on-demand with a button in post editor.
    
    **Pros:**
    - Full control over when to generate
    - No unexpected AI usage
    - Can regenerate if needed
    
    **Cons:**
    - Requires manual action
    - Easy to forget
    
    **Best For:** Cost-conscious users
  </Tab>
  
  <Tab title="Bulk Action">
    ### Bulk Generate
    
    Tag multiple posts at once from Posts list.
    
    **Pros:**
    - Process many posts quickly
    - Great for existing content
    - One-time AI usage
    
    **Cons:**
    - Not automatic
    - Must select posts manually
    
    **Best For:** Retroactive tagging
  </Tab>
</Tabs>

## Tag Strategy Options

<AccordionGroup>
  <Accordion title="Reuse Existing Tags Only" icon="recycle">
    **Configuration:**
    - Use Existing Tags: **Yes**
    - Allow New Tags: **No**
    
    **Behavior:**
    AI only selects from your current WordPress tags.
    
    **Best For:**
    - Mature blogs with established taxonomy
    - Preventing tag proliferation
    - Maintaining strict tag control
    
    **Example:** Site has 50 predefined tags; AI picks 5 most relevant.
  </Accordion>
  
  <Accordion title="Hybrid Approach (Recommended)" icon="balance-scale">
    **Configuration:**
    - Use Existing Tags: **Yes**
    - Allow New Tags: **Yes**
    
    **Behavior:**
    AI prefers existing tags but creates new ones when needed.
    
    **Best For:**
    - Growing blogs
    - Evolving topics
    - Balanced taxonomy management
    
    **Example:** AI uses 4 existing tags + creates 1 new relevant tag.
  </Accordion>
  
  <Accordion title="Always Create New" icon="sparkles">
    **Configuration:**
    - Use Existing Tags: **No**
    - Allow New Tags: **Yes**
    
    **Behavior:**
    AI creates fresh tags for every post.
    
    **Best For:**
    - New blogs starting from scratch
    - Highly diverse content
    - Maximum coverage
    
    **Example:** Every post gets 5 unique, content-specific tags.
  </Accordion>
</AccordionGroup>

## Manual Tag Generation

Even with auto-tagging enabled, you can manually generate tags:

### In Post Editor

<Steps>
  <Step title="Open Post">
    Edit any post in WordPress editor
  </Step>
  
  <Step title="Find Generatify Panel">
    Look for **Generatify** meta box in sidebar
  </Step>
  
  <Step title="Click Generate Tags">
    Click **Generate Tags with AI** button
  </Step>
  
  <Step title="Review Tags">
    AI-generated tags appear in the Tags field
  </Step>
  
  <Step title="Edit if Needed">
    Modify or remove tags before saving
  </Step>
</Steps>

### Bulk Tag Generation

<Steps>
  <Step title="Go to Posts List">
    Navigate to **Posts → All Posts**
  </Step>
  
  <Step title="Select Posts">
    Check the posts you want to tag
  </Step>
  
  <Step title="Choose Bulk Action">
    Select **Generate AI Tags** from Bulk Actions dropdown
  </Step>
  
  <Step title="Click Apply">
    Click **Apply** to start generation
  </Step>
  
  <Step title="Wait for Completion">
    Progress bar shows tagging status
  </Step>
</Steps>

<Info>
**Bulk Limits:** Process up to 100 posts at once. For larger batches, repeat the process.
</Info>

## Advanced Customization

### Custom Tag Prompts

Customize the AI prompt to match your tagging style:

```
Navigate to: Generatify → Settings → Automations → Auto Tags → Advanced

Default Prompt:
"Analyze this blog post and generate 5 relevant tags. 
Use existing tags when possible: {existing_tags}."

Custom Example:
"Generate 5 SEO-optimized tags for this tech blog post. 
Focus on programming languages, frameworks, and developer tools. 
Prefer existing tags: {existing_tags}."
```

### Hook for Developers

Modify tags before they're saved:

```php
add_filter('generatify_auto_tags', function($tags, $post_id) {
    // Remove tags longer than 20 characters
    $tags = array_filter($tags, function($tag) {
        return strlen($tag) <= 20;
    });
    
    // Always add site-specific tag
    $tags[] = 'DevBlog';
    
    return $tags;
}, 10, 2);
```

<Note>
See [Developer Hooks](/developer/hooks-filters) for more customization options.
</Note>

## Tag Quality Tips

<AccordionGroup>
  <Accordion title="Provide Context" icon="book">

    - Write clear, descriptive titles
    - Include key terms in first paragraph
    - Use structured headings

  </Accordion>
  
  <Accordion title="Seed with Existing Tags" icon="seedling">
    Give AI examples to follow:
    - Manually tag first 10-20 posts
    - Establish your tagging patterns

    - Create tag descriptions
    
    AI learns from your existing taxonomy.
  </Accordion>
  
  <Accordion title="Review and Refine" icon="edit">

    - Review auto-generated tags initially

  <Accordion title="Set Realistic Limits" icon="sliders">

## Troubleshooting

<AccordionGroup>
  <Accordion title="Tags Not Generating">
    **Symptoms:** No tags appear after publish/save
    
    **Solutions:**
    1. Check that Auto Tags is enabled
    2. Verify AI provider is connected
    3. Ensure post type is included in settings
    4. Check post isn't in excluded category
    5. Look for JavaScript errors in console
    6. Test with manual generation button
  </Accordion>
  
  <Accordion title="Irrelevant Tags">
    **Symptoms:** AI generates off-topic tags
    
    **Solutions:**
    1. Improve post content clarity
    2. Add excerpt with topic focus
    3. Adjust number of tags (fewer = more focused)
    4. Switch to better AI model (GPT-4o Mini)
    5. Customize AI prompt for your niche
    6. Seed with more existing relevant tags
  </Accordion>
  
  <Accordion title="Too Many New Tags Created">
    **Symptoms:** Tag library growing uncontrollably
    
    **Solutions:**
    1. Enable "Use Existing Tags" preference
    2. Disable "Allow New Tags"
    3. Regularly merge similar tags
    4. Review and clean up tag library
    5. Set up custom prompt limiting new tags
  </Accordion>
  
  <Accordion title="Slow Tag Generation">
    **Symptoms:** Takes 10+ seconds to generate
    
    **Solutions:**
    1. Switch to faster model (Gemini Flash, Claude Haiku)
    2. Check AI provider status
    3. Reduce post content length analyzed
    4. Check server performance
    5. Disable on-save trigger (use on-publish only)
  </Accordion>
  
  <Accordion title="API Quota Exceeded">
    **Symptoms:** Error message about limits
    
    **Solutions:**
    1. Switch to Google Gemini (higher free tier)
    2. Disable auto-run, use manual trigger
    3. Exclude less important post types
    4. Upgrade AI provider plan
    5. Use bulk generation less frequently
  </Accordion>
</AccordionGroup>
